# 移动端

前端在移动端开发中占有重要地位，本质是因为前端的运行时——浏览器是跨平台的，另一方面前端开发效率高、成本低、从业者多，也吸引了更多人将前端应用到移动端。

前端应用与原生应用最大的区别是前端只能运行在浏览器上，权限和能力依赖于浏览器、十分有限，为了实现更多能力和媲美原生的用户体验，需要封装移动端框架来集成原生能力。

PC 端也有类似的尝试——Electron。

按照原生能力集成的程度，前端移动端开发可以分为 3 类：

1. H5：普通的移动端网页，相比 PC 端适配了移动端布局和交互，不依赖原生能力
   - 不依赖原生能力的套壳 App 也归为此类
2. 小程序：最早由微信提出，提供了基于宿主 App 的部分原生能力，语法需要转译
3. Web App：移动端框架封装和提供了绝大部分原生能力，使开发者的能力更加全面
   - 更多的是 Hybrid App，但此处只关注前端部分
